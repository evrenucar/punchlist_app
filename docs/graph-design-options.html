Graph redesign — four directions

Sketches with real items from this week. Every option keeps your three requirements: arrow-key browsing, expand/compact per node like the task board, and granular children. They differ in what the LAYOUT itself tells you. Patterns borrowed from: git commit graphs (A), mind-map/org trees (B), CI pipeline views (C), Obsidian-style canvas (D).

A · The spine — time runs down, like a git log

Nails: "what was added last" (always the top) and "what's next" (right under the glow). Verticality IS the layout. Tradeoff: parallel work shows as interleaved rows, not side-by-side branches.
