Updating your downloaded Punchlist

Punchlist is a single HTML file. If you downloaded it and open it from your own disk, here is how to move to a newer build without losing your board.

How your board is stored

Punchlist keeps your board in the browser's local storage, tied to the exact location and the way you open the file. Same folder, same filename, opened the same way (the same double-click, shortcut, or bookmark) means the same saved board. Change the folder or the filename and the browser treats it as a new place, so it opens with an empty board. Nothing is deleted in that case; the old board is still attached to the old location.

So the whole trick to updating in place is simple: put the new file where the old one was, under the same name, and open it the way you always do.

Update in five steps

  1. Find your current file and note two things: the folder it sits in, and its exact filename (for example task-board.html). You will reuse both.
  2. Back it up first (details below). Open your current board and click Export JSON to save a copy of your data somewhere safe.
  3. Open the GitHub Releases page and download the newest build.
  4. Put the new file in the same folder as the old one, under the same filename, replacing it. If your browser saved it under a different name (like task-board(1).html), rename it to match the old file exactly before you replace it.
  5. Open it the same way you always have (same double-click from that folder, same shortcut, or same bookmark). Your saved board is there, now running the newer build.
If the new file opens empty, you almost certainly opened it from a different folder or under a different name. Your board is not gone. Move or rename the file back to the old location and open it again, or use the import step below.

Safety net: Export JSON and Import JSON

The JSON export is a full, lossless copy of your board. It is worth doing before you replace anything, and it always works even if the file path changes.

  1. Before updating, open your current board and click Export JSON. Save the file somewhere outside the app's folder.
  2. If the updated file looks wrong or empty, open it and click Import JSON, then pick the file you just saved. Your board comes back.

Using the hosted version instead

The hosted app at evrenucar.github.io/punchlist_app is always the latest build automatically. If you use it there, none of this applies; there is nothing to update.
