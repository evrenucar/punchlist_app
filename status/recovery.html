Punchlist · data recovery

Get your board back

Your board did not vanish. A stale device overwrote it through GitHub sync, and every sync keeps a full backup as a commit. The good version is still in your repo. Here is how to pull it back.

The short version Open your sync repo on GitHub, look at the history of punchlist-board.json, grab the version from just before the bad sync, and Import it into the app. Ten minutes, no data lost.

What actually happened

Sync stores your whole board as one file in a private repo. When two devices have both changed, the app currently lets the local device win, with no check for which board is newer. A device that was last used 5 days ago came back online, still counting as changed, and pushed its old board over your current one. Your other devices then pulled that old board down.

asus_laptop (this device)today · 10:12
device a07ytoday · 04:22
device hkth ← the culprit5 days ago · 09:00

That 5-days-ago device is the one whose old board won. The other two read “today” only because they saved again after pulling the old board down.

Before you touch anything Do not let the stale device (hkth) sync again until you have restored and the fix is in, or it can overwrite you a second time. Easiest: keep that device closed, or turn its sync toggle off (Settings → Sync) for now.

Recover the board

  1. Open your sync repo on GitHub

    It is the private repo shown in the app under Settings → Sync → Repository (evrenucar/punchlist-…). Open it in a browser where you are signed in to GitHub.

  2. Open the board file and its history

    Click the file punchlist-board.json, then click History (top right of the file view). You will see one commit per sync.

    punchlist-board.jsonHistory ›
    punchlist sync (load, hkth)today · the bad push
    punchlist sync (edit, asus_laptop)today · ← your good board
    punchlist sync (edit, a07y)today, earlier

  3. Find the last good commit

    The bad one is the sync from the stale device, message like punchlist sync (load, hkth), timed when your board reverted. The commit right above it in time (the one just before) is your good board. Its message names your real device, like punchlist sync (edit, asus_laptop).

    • Not sure which is which? Click a commit to preview that version of the file and scan the task text. Pick the one that has your recent work.

  4. Download that good version

    Click the good commit, then open punchlist-board.json at that point, click Raw, and save the page (Ctrl+S) as a .json file.

    • On GitHub a file view has a Raw button top right. Right-click it → Save link as, or open it and save.

  5. Back up the current (broken) board first

    In the app, export the board you have now before overwriting it, just as a safety net. Menu → Export JSON. Keep that file somewhere.

  6. Import the good version

    In the app on your main device, use Import JSON and pick the file you downloaded in step 4. Your board comes back. Your images come with it: image files live separately in the repo and are never deleted, so old references still load.

Then let it spread After importing on your main device, that device pushes the restored board back up, and your other devices pull it. Give it a minute, then check Settings → Sync shows “Synced” and the board looks right on each device.
One order-of-operations note Until the fix ships, the same thing could happen again if any stale device syncs while dirty. Safest sequence: I ship the guard first, then you restore, so nothing can re-clobber. If you would rather restore right now, just keep hkth offline until the fix lands.
